Setup Guide

One-time Cloudflare setup — takes about 20 minutes

⚙️ Step-by-Step Cloudflare Setup

  • Create a GitHub repo — Upload all your website files (index.html, pages/, admin/, cms-data.json). Make it public or private — both work.
  • Connect to Cloudflare Pages — Go to dash.cloudflare.com → Workers & Pages → Create → Pages → Connect to Git → Select your repo. Build command: leave blank. Output dir: / (root). Deploy!
  • Create R2 Bucket for images — In Cloudflare dashboard → R2 → Create Bucket → name it prime-trust-images. Then go to Settings → Public Access → Enable. Copy the public domain (e.g. pub-xxxx.r2.dev).
  • Create KV Namespace — Workers & Pages → KV → Create Namespace → name it PRIME_TRUST_KV. Copy the namespace ID.
  • Deploy the Worker — Install Wrangler: npm install -g wrangler. Then wrangler login. Paste KV namespace ID into wrangler.toml. Run wrangler deploy.
  • Set Admin Password — Run: wrangler secret put ADMIN_PASSWORD and type your password. Keep this safe!
  • Upload CMS data — Come back to this admin panel. Enter your Worker URL + password. Go to any section and click Save Changes. The site will start reading live data!
  • Update Worker URL in admin/index.html — Open admin/index.html and set the default Worker URL so you don't have to type it every time.

📋 wrangler.toml Template

# wrangler.toml
name = "prime-trust-worker"
main = "functions/worker.js"
compatibility_date = "2024-01-01"

[[kv_namespaces]]
binding = "PRIME_TRUST_KV"
id = "YOUR_KV_ID_HERE"

[[r2_buckets]]
binding = "PRIME_TRUST_IMAGES"
bucket_name = "prime-trust-images"
💡 Image Hosting Note: After enabling R2 public access, update the publicUrl in functions/worker.js to use your R2 bucket domain instead of the placeholder.
